1. Probar backend

Entrá a backend:

npm run dev

Debe mostrar:

Servidor corriendo en puerto 3001
2. Probar frontend

Entrá al frontend:

npm run dev

Abrí el navegador y probá el chat.

Pruebas por tema
1. Chat básico

Mensaje:

Hola, ¿cómo estás?

Debe responder normalmente.

Probás:

Frontend
Backend
Gemini/mock
Respuesta básica
2. RAG

Mensaje:

¿Qué hábitos ayudan a dormir mejor?

Revisá en debug:

Contexto RAG recuperado
Chunks recuperados

Debe aparecer contenido desde Qdrant.

Probás:

Embeddings
Qdrant
recuperarContextoRag()
Nodo recuperarRag
3. Memoria conversacional

Mensaje 1:

Mi objetivo es mejorar mi descanso.

Mensaje 2:

¿Cuál es mi objetivo?

Debe recordar por historial reciente.

Probás:

historial
memoria de sesión
4. Memoria persistente

Mensaje:

Recordá que mi objetivo es bajar de peso.

Después reiniciá el chat con el botón y preguntá:

¿Qué recordás de mi objetivo?

Debe aparecer algo en:

Memoria persistente recuperada

Probás:

memory-service.js
Qdrant memoria_agente
guardarMemoria()
recuperarMemoria()
Nodo recuperarMemoria
5. Planning

Mensaje:

Peso 80 kg, mido 1.75 y quiero una rutina principiante urgente.

Revisá:

Plan del agente

Debe aparecer algo tipo:

{
  "objetivo": "...",
  "pasos": ["...", "..."],
  "herramientas_posibles": ["calcularIMC", "generarRutina"]
}

Probás:

nodoPlanning
planificación
6. Tool Calling

Mensaje:

Peso 80 kg y mido 1.75.

Revisá:

observaciones

Debe aparecer:

{
  "funcion": "calcularIMC",
  "argumentos": {
    "peso": 80,
    "altura": 1.75
  },
  "ok": true
}

Probás:

tools.js
ejecutarHerramienta()
validación de argumentos
7. ReAct Loop

Mensaje:

Peso 80 kg, mido 1.75 y quiero una rutina principiante urgente.

Revisá:

ReAct Loop

Debe verse:

pensamiento
accion
observaciones
iteracion

Probás:

pensar
ejecutarHerramienta
volver a pensar
generarRespuestaFinal
8. Streaming

Mensaje:

Armame una respuesta completa sobre cómo mejorar mi descanso.

Debe verse la respuesta escribiéndose de a poco.

Probás:

/api/chat/stream
event: chunk
event: done
fetch streaming en App.jsx
9. Checkpointing

Mandá cualquier mensaje y revisá que no falle.

Checkpointing no se ve tanto visualmente, pero se activa porque usás:

MemorySaver

y:

thread_id: usuarioId

Probás:

checkpointer
thread_id
estado persistido en RAM
10. Modo mock

En .env:

USE_MOCK_AI=true

Mandá:

Probando modo mock

Debe responder sin consumir cuota de Gemini.

Después, para volver al modo real:

USE_MOCK_AI=false

Probás:

modo desarrollo sin gastar requests
